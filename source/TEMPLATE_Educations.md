# Education Entry Template

This template ensures consistency across all education-related markdown files in the `Educations` directory.

## Naming Convention

Files should be named according to the following pattern:
`YYYY_Degree_Institution_Field.md`

- **YYYY**: The start year of the program.
- **Degree**: The degree abbreviation (e.g., BASc, MASc, PhD).
- **Institution**: Shortened institution name (e.g., UoT, UBC).
- **Field**: Shortened field of study (e.g., ChemEng, MineEng).

---

## Template Structure

```markdown
## [Full Degree Name]

**Workplace**
[Institution Name](URL), City, Country.

**Timeline**
YYYY Month. - YYYY Month.

**Project**
[Project Title], with [Supervisor Name](Google Scholar or Profile URL).

**Project Goal**
[One or two sentences describing the primary objective of the project.]

**Project Summary** (Optional)
[A more detailed summary if the Project Goal and Tasks Performed don't cover everything.]

**Tasks Performed**
- [Task 1 Description]
- [Task 2 Description]
- [Task 3 Description]
- [Task 4 Description]
```

## Instructions for Use

1. **Header**: Use a second-level header (`##`) for the degree name.
2. **Workplace**: include a markdown link to the institution's official website.
3. **Timeline**: Use the format `YYYY Month. - YYYY Month.` (e.g., `2007 Sept. - 2011 Sept.`).
4. **Project**: Clearly state the project title. Include the supervisor's name with a link to their academic profile (e.g., Google Scholar).
5. **Bold Key Terms**: Use **bold text** within the Project Goal and Tasks Performed to highlight key technologies, methodologies, or outcomes (e.g., **Aspen HYSYS**, **Finite Element Modeling**, **retrofitting**).
6. **Task List**: Use a bulleted list for specific technical tasks and achievements.
